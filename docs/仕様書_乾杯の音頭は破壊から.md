# 乾杯の音頭は破壊から - ゲーム仕様書

## 1. プロジェクト概要

### 1.1 ゲーム概要
- **タイトル**: 乾杯の音頭は破壊から (Cheers Starts with Destruction)
- **ジャンル**: 体感型アクション / 乾杯バトル
- **プラットフォーム**: Mac (Unity)
- **入力デバイス**: 実物ビールジョッキ + M5系センサー（加速度センサー、6軸IMU、マイク）

### 1.2 ゲームコンセプト
プレイヤーは実物のビールジョッキを振り、NPCと1対1の乾杯勝負を行う。タイミングよく乾杯できれば相手のグラスを破壊し、失敗すれば自分のグラスがダメージを受ける。自分のグラスが壊れるまで何人のNPCを倒せるかを競う「百人抜き」形式のエンドレスモード。

### 1.4 ステージ設定
- **現行**: 居酒屋（1次会）
- 背景・世界観はデザイン担当が作成
- 将来的にステージ追加の可能性あり（拡張時に選択画面を実装）

### 1.3 開発体制
| 担当 | 範囲 |
|------|------|
| システム担当（自分） | Unityゲームロジック、UI、ゲームフロー |
| センサー担当 | センサーハードウェア、センサー↔PC通信 |
| デザイン担当 | キャラクターデザイン、3Dモデル |

## 3. システムアーキテクチャ

### 3.1 設計方針
- **センサー入力の抽象化**: インターフェースを定義し、モック実装でセンサーなしでもテスト可能にする

## 7. 乾杯バトルシステム

### 7.1 バトルフロー

```
1. NPCが乾杯の構えを取る
2. カウントダウン表示（「3, 2, 1, 乾杯！」）
3. プレイヤーがジョッキを振る
4. 判定処理
5. 結果に応じた演出・ダメージ処理
```

### 7.2 タイミング判定（リズムゲーム方式）

画面上にタイミングガイドを表示し、ガイドに合わせてジョッキを振ることで判定を行う。

```
[タイミングガイドUI]

    ◄── ターゲットゾーン ──►
         |  Perfect  |
      |    Great    |
   |      Good      |
───●────────────────────●───► 時間軸
   │                    │
  判定開始            判定終了
   
   ★ ←── インジケーター（移動してくる）
```

```csharp
public enum TimingGrade
{
    Perfect,    // 最高判定
    Great,      // 良判定
    Good,       // 可判定
    Miss        // ミス（タイミング外）
}

public enum CheersResult
{
    Whiff,          // スカ（タイミング早すぎ/速度不足）
    Victory,        // 勝利（相手のグラス破壊）
    Defeat,         // 敗北（自分のグラスにダメージ）
    Draw,           // 引き分け（両者破壊）
    SelfDestruct    // 自爆（角度不正）
}
```

### 7.3 攻撃力計算

攻撃力は**タイミング精度**と**声量**の2要素で決定する。

```csharp
public class AttackCalculator
{
    public float CalculateAttackPower(TimingGrade timing, float voiceVolume)
    {
        float timingMultiplier = timing switch
        {
            TimingGrade.Perfect => 1.5f,
            TimingGrade.Great   => 1.2f,
            TimingGrade.Good    => 1.0f,
            TimingGrade.Miss    => 0.0f,
            _ => 0.0f
        };
        
        // 声量ボーナス（0.0〜1.0の正規化値 → 0%〜50%のボーナス）
        float voiceBonus = 1.0f + (voiceVolume * 0.5f);
        
        return baseAttackPower * timingMultiplier * voiceBonus;
    }
}
```

**攻撃力の影響**:
- 攻撃力がNPCの防御閾値を超えれば **Victory**
- 下回れば **Defeat**
- ギリギリの場合は **Draw**

### 7.4 判定ロジック

```csharp
public class BattleManager : MonoBehaviour
{
    public CheersResult JudgeCheers(CheersInputData input, VoiceInputData voice, NPCData npc)
    {
        // 1. 角度チェック（自爆判定）
        if (IsAngleBad(input.Angle))
            return CheersResult.SelfDestruct;
        
        // 2. タイミング判定
        TimingGrade timing = JudgeTiming(input.Timestamp);
        if (timing == TimingGrade.Miss)
            return CheersResult.Whiff;
        
        // 3. 攻撃力計算
        float attackPower = CalculateAttackPower(timing, voice.Volume);
        
        // 4. 勝敗判定
        float npcDefense = npc.DefenseThreshold;
        
        if (attackPower > npcDefense * 1.2f)
            return CheersResult.Victory;
        else if (attackPower > npcDefense * 0.8f)
            return CheersResult.Draw;
        else
            return CheersResult.Defeat;
    }
}
```

### 7.5 ダメージ計算

| 結果 | プレイヤーへのダメージ | NPCへの効果 |
|------|----------------------|-------------|
| Whiff | 0 | なし（仕切り直し） |
| Victory | 0 | 即撃破 |
| Defeat | 中ダメージ | なし |
| Draw | 小ダメージ | 即撃破 |
| SelfDestruct | 大ダメージ | なし |

※具体的なダメージ値はバランス調整で決定
